# Архитектура SkySweep32

## Обзор

SkySweep32 использует многозадачную архитектуру FreeRTOS на двухъядерном ESP32. Каждый модуль работает в своей задаче, обмен через общие структуры данных, защищённые мьютексами.

## Задачи FreeRTOS

```
Core 0 (Протокольное ядро)         Core 1 (Прикладное ядро)
┌─────────────────────┐            ┌─────────────────────┐
│ taskRFScanning      │            │ taskDisplayUpdate    │
│ Приоритет: 3        │            │ Приоритет: 1         │
│ • Опрос CC1101      │            │ • Обновление OLED    │
│ • Опрос NRF24       │            │ • Полосы сигнала     │
│ • Опрос RX5808      │            └─────────────────────┘
│ • Парсинг протоколов│            ┌─────────────────────┐
│ • Оценка угроз      │            │ taskWebServer       │
│ • ML-классификация  │            │ Приоритет: 2         │
└─────────────────────┘            │ • WiFi AP           │
┌─────────────────────┐            │ • HTTP + WebSocket  │
│ taskRemoteID        │            └─────────────────────┘
│ • BLE-сканирование  │            ┌─────────────────────┐
└─────────────────────┘            │ taskGPS             │
```

## Шина SPI

Все устройства SPI делят одну шину через SPIManager с мьютексом:

```
Устройства SPI:
├── CC1101    CS=GPIO 5   [Standard+]
├── NRF24L01+ CS=GPIO 15  [Base+]
├── RX5808    CS=GPIO 13  [Standard+]
├── LoRa      CS=GPIO 26  [Pro]
└── SD Card   CS=GPIO 27  [Pro]
```

## Условная компиляция

```
TIER_BASE     → NRF24, OLED, WebServer, RemoteID
TIER_STANDARD → Base + CC1101, RX5808
TIER_PRO      → Standard + GPS, SD Card, LoRa
```

## Использование памяти

| Компонент | Оценка RAM |
|-----------|------------|
| Задачи FreeRTOS | ~35 КБ |
| Веб-дашборд (PROGMEM) | ~15 КБ (Flash) |
| RF буферы | ~2 КБ |
| Массив Remote ID | ~4 КБ |
| **Итого** | **~60 КБ из 520 КБ** |
